# Shift at Midnight Launch Scope

## Launch Slice

Build a fast-test static guide/status site with 12 routes:

```text
/
/release-date/
/demo/
/platforms/
/beginner-guide/
/customer-checks/
/joes-diner-newsletter/
/sources/
/about/
/editorial/
/contact/
/privacy/
```

## Required Components

- homepage status board;
- release conflict tracker;
- demo availability table;
- platform capability table;
- source confidence matrix;
- spoiler-safe customer-check workflow;
- Joe's Diner newsletter source-status FAQ;
- NOW / PARTIAL / HOLD boundary panels;
- source link panel with official/platform/press/video/community labels.

## Content Boundaries

Can publish:

- current Steam status if rechecked on build day;
- Steam demo existence;
- Xbox store existence and visible capability facts;
- release-date conflict/status timeline;
- official source links;
- spoiler-safe explanation of the game premise and workflow;
- Trends-derived demand notes only in internal planning or generic public wording.

Must HOLD:

- exact release date unless current official/platform source confirms it;
- exact Joe's Diner newsletter answer unless official or hands-on verified;
- exact doppelganger/customer tells;
- walkthroughs;
- routes/maps/secrets/endings;
- survival-phase exact tactics;
- unsupported PS5 claim;
- copied screenshots, trailer frames, thumbnails, logos, or press images.

## Validation Before Build Handoff

Before asking Claude Code or another agent to build:

1. Reopen Steam store and Xbox store.
2. Check Steam news for latest release/status post.
3. Confirm no public page in the plan requires unverified walkthrough or spoiler facts.
4. Confirm the build prompt says "no images unless self-made or rights-cleared."
5. Confirm public repo isolation rules.

## Acceptance Criteria

- Site feels like a useful guide/status hub, not a thin date page.
- Each content page has at least one structured component.
- Each claim has source labels or visible uncertainty.
- HOLD boundaries are visible to users.
- No copyrighted media is downloaded or embedded.
- No competitor guide or YouTube gameplay content is copied.
- Local build/check/validate must pass before deploy.
